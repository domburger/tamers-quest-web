Monster generation — settings by phase

Each pipeline phase has its own model + sampling. Idea → a concept; Attributes → stats, the 4 attacks & a visual description; Builder → the from-scratch shape model. Blank a field & save to reset. Applied live.
1 · Idea — a 2–4 word concept
2 · Attributes — stats, 4 attacks, visual description
3 · Builder — draws the creature from scratch as free-form SVG (needs a capable model)

Monster prompts (by phase)

The system/user prompt for each phase. Clear a field & save to reset. Keep the {hints} / {idea} / {monster} placeholders.

Schema field descriptions

Structured-output field descriptions sent to the LLM to steer each generated field. {stat} is substituted per stat.

Generation hub — test generation (no save)

Run a TEST generation with the CURRENT settings (models / temps / prompts / schema descriptions above) and preview the raw output. Nothing is written to the live pool — this is a dry run (TQ-213).
